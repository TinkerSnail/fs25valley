## Goal

Make Walter (base-game GRANDPA NPC) walk forward along a waypoint loop, FACING his
direction of travel, with a walk animation — no gliding sideways, no rotation twitching.
Movement works. The unsolved problem is rotation: setting his facing causes visual twitching.

## Architecture

Walter is the base-game GRANDPA NPC. He does NOT own his character node — `g_npcManager` does.
- `g_npcManager:getNPCByName("GRANDPA")` → grandpa object with `.node`, `.x`, `.y`, `.z`, `.rotY`, `.spot`, `.playerGraphics`
- `grandpa.playerGraphics.graphicsRootNode` = visual mesh root, parent = WORLD ROOT (no useful intermediate parent to rotate instead)
- `grandpa.node` = separate subtree, NOT a parent of graphicsRootNode
- `g_npcManager:update()` calls `playerGraphics:update(dt)` every frame on GRANDPA
- Our `WalterWalker:update(dt)` runs AFTER g_npcManager via the FSBaseMission.update hook
- **ALL node IDs are runtime handles that CHANGE EVERY SESSION. Always read them from fields; never hardcode.**

## ROOT CAUSE (after R13 — measured, this is THE finding)

the bones, not the node. So rotating graphicsRootNode was always the wrong lever for FACING.
